### 第二步：添加要改名的文件

把文件直接拖拽进 ReNamer 的文件区，或者点击工具栏上的「添加文件」按钮，支持同时添加多个文件夹。

### 第三步：在规则区添加改名规则

点击规则区上方的「➕ 添加」，弹出规则设置窗口。规则类型很多，常用的有：

- **插入**：在文件名指定位置插入文字、序号、日期、元标签
- **删除**：删除指定位置的字符，或删除特定字符串
- **替换**：把某段文字换成另一段，支持正则表达式
- **大小写**：统一转成全大写、全小写、或首字母大写
- **序列化**：批量加序号，可以自定义起始值和步进
- **扩展名**：单独修改文件后缀

多个规则可以叠加，按列表顺序依次执行。

### 第四步：预览结果，确认无误后执行

添加规则后，文件区会实时显示改名预览。仔细看一下「新名称」列，没问题了点击右上角「✔ 重命名」。

### 第五步：保存为规则集（预设）

## 正则表达式在 ReNamer 里怎么用？

这是 ReNamer 比普通改名工具强的地方之一。

在添加「替换」规则时，勾选「使用正则表达式」，就可以用正则模式匹配文件名中的任意字符组合。举几个实际场景：

**场景 1：删除文件名里所有括号和括号内容**
匹配模式：`\(.*?\)` 或 `\[.*?\]`，替换为空

**场景 2：提取日期，重新排列顺序**
比如把「20240315」这样的日期改成「2024-03-15」，可以用捕获组：
匹配 `(\d{4})(\d{2})(\d{2})`，替换为 `$1-$2-$3`

**场景 3：批量去掉文件名里的多余空格**
匹配 `\s+`，替换为单个空格或直接替换为空

ReNamer 完整支持标准正则语法，对习惯命令行改名的用户来说，用图形界面调试正则规则会轻松很多。
